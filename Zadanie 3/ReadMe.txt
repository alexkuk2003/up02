Краткое описание проекта:
Регистрируемся, создаем сайт со страницами на html, сохраняем сайт, скачиваем ZIP-архив сайта и даже можем его посмотреть,
что отражено на скриншотах.

Как запускал, чтобы сделать скриншоты:

1) локально на на компе запускаем Node.js -> Node.js command promt
2) переходим в папку проекта: # cd C:\UP02\Zadanie 3\
3) в этой папке лежит движек на js: server.js.
4) а в папке C:\UP02\Zadanie 3\public лежат фалы проекта, написанные в VSCode: index.html, generator.html, dashboard.html. style.css
5) Выполним:
   npm init -y
   npm install 
   npm install express sqlite3 body-parser body-parser jsonwebtoken uuid multer archiver formidable cookie-parser bcrypt formidable cors uuid nodemailer

 6) package.json - конфиг создается командой npm init -y.
    Это основной конфигурационный файл Node.js проекта (название проекта, версия, описание, скрипты и т.д., всё по умолчанию благодаря -y).
    package-lock.json - конфиг создается командой npm install (дерево зависимостей, которое было установлено).
    node_modules - это папка, в которой хранятся все установленные Node.js модули (пакеты) и их зависимости.

7) После этого запускаем движек в cd C:\UP02\Zadanie 3\ командой # node server.js
   
8) Я приложил сам движек server.js скриншоты в корне проекта и дизайн-файлы в папке public.
   Скриншоты я сделал после запуска проекта. Деплой локальный - на компе. Скоро задеплою на vps.

9) Название скриншотов с результатами работы проекта BobbyHost:

1.1. Вкладка регистрации на главной странице сервиса.jpg
1.2. Вкладка входа на главной странице сервиса.jpg

3. После нажатия кнопки Создать новый сайт.jpg

6. Сайт сгенерирован в папку alex в папке exports.jpg
7. Главная страница сайта.jpg


Описание фичей (установленных пакетов):

1.  express: Основной фреймворк для создания веб-сервера.
2.  body-parser: Для парсинга входящих HTTP-запросов (особенно JSON и urlencoded).
3.  jsonwebtoken: Для работы с JSON Web Tokens (JWT) для аутентификации.
4.  fs.promises: Встроенный модуль Node.js для работы с файловой системой (используется асинхронно).
5.  path: Встроенный модуль Node.js для работы с путями к файлам.
6.  uuid: Для генерации уникальных идентификаторов (UUID).
7.  archiver: Для создания ZIP-архивов.
8.  formidable: Для обработки загрузки файлов (multipart/form-data).
9.  cookie-parser: Для парсинга куки из HTTP-заголовков.
